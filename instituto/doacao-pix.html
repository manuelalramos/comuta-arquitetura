Como usar com seguranca no Wix

O HTML sozinho pode mostrar a cobranca, mas a geracao do PIX dinamico precisa acontecer fora desta pagina, em um endpoint seguro.

  • Guarde token, client secret e credenciais do banco ou gateway apenas no backend.
  • Faca esta pagina chamar um endpoint HTTPS que crie a cobranca e devolva somente os dados necessarios para exibicao.
  • Defina expiracao curta para o QR Code e regenere quando vencer.
  • Valide a confirmacao do pagamento por webhook no servidor, e nao pela tela do navegador.
  • Se usar Velo no Wix, mantenha os segredos no Secrets Manager e execute a chamada ao PSP no backend.
Endpoint esperado nesta pagina: troque PIX_API_URL no script por um endpoint seu ou do provedor. O endpoint deve retornar JSON com qrCodeImage, pixCode, donationId, expiresAt e opcionalmente status.
